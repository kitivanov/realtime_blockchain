# realtime_blockchain

Потоковый индексер (collector) блокчейн-событий на Go.

Проект получает события из сети Ethereum в режиме реального времени через WebSocket от Alchemy API,
дополняет их информацией с помощью JSON-RPC запросов,
декодирует события стандарта ERC-20 и преобразует их в структурированные данные, пригодные для дальнейшей обработки и анализа.


## Основные возможности

* получение событий сети Ethereum через WebSocket
* дедупликация входящих событий
* пакетная обработка (batching)
* выполнение batch JSON-RPC запросов
* параллельное получение транзакций и receipt
* объединение результатов независимых запросов
* декодирование событий ERC-20 Transfer
* обогащение данных информацией о токенах
* получение актуальной стоимости токенов через CoinGecko
* формирование структурированного результата

На текущем этапе разработки результат обработки выводится в консоль. В дальнейшем планируется отправка обработанных событий в Kafka.


## Архитектура проекта

Проект реализован в виде пайплайна, где каждый компонент выполняет одну задачу и взаимодействуют через каналы Go.

```
Ethereum WebSocket
        │
        ▼
 WSLogCollector
        │
        ▼
 Deduplicator
        │
        ▼
    Batcher
        │
        ▼
 FetcherFanOut
   ├───────────────┐
   ▼               ▼
ReceiptFetcher  TransactionFetcher
   └──────┬────────┘
          ▼
        Joiner
          │
          ▼
    ERC20Decoder
          │
          ▼
      Enricher
          │
          ▼
      Console Output
```

## Описание каждого узла пайплайна

### 1. Получение событий

Подключение к Ethereum выполняется через WebSocket Alchemy API.

Получаются события типа `Transfer` стандарта ERC-20.


### 2. Deduplicator

Удаляет повторяющиеся транзакции и хранит недавно обработанные хэши в памяти с использованием TTL.


### 3. Batcher

Объединяет несколько событий в один пакет.

Пакет отправляется либо:

* при достижении заданного размера
* либо по истечении заданного интервала времени

Это сделано для батчевания и уменьшения RPC-запросов.


### 4. FetcherFanOut

Разделяет поток батчей на две независимые ветви:

* получение Transaction
* получение Receipt

Обе ветви работают параллельно.


### 5. ReceiptFetcher

Получает данные транзакций методом

```
eth_getTransactionReceipt
```

с использованием batch JSON-RPC.


### 6. TransactionFetcher

Получает информацию о транзакциях методом

```
eth_getTransactionByHash
```

также пакетными запросами.


### 7. Joiner

Объединяет Transaction и Receipt одной транзакции.

После получения обеих частей данные передаются дальше по конвейеру.


### 8. ERC20Decoder

Извлекает из Receipt события ERC-20 Transfer. Формируется единая модель события.


### 9. Enricher

Обогащает необходимыми данными:

* названием токена
* символом токена
* количеством знаков после запятой
* количеством токенов в удобном формате
* стоимостью в USD (при наличии данных)


### 10. Вывод результата

На текущем этапе результат сериализуется в JSON и выводится в консоль.


## Особенности реализации

В проекте используются следующие подходы:

* конкурентная обработка с помощью Goroutines
* обмен данными через Channels
* кэширование данных о токенах
* пакетные JSON-RPC запросы для минимизация количества сетевых обращений


## Запуск проекта

### 1. Клонирование

```bash
git clone <repository>
cd realtime_blockchain
```

### 2. Настройка переменных окружения

Создать файл `.env`.

Пример:

```env
AlchemyKey=YOUR_KEY
AlchemyURLWS=wss://eth-mainnet.g.alchemy.com/v2/
AlchemyURLHTTP=https://eth-mainnet.g.alchemy.com/v2/

TransferTopic=0xddf252ad1be2c89b69c2b068fc378daa952ba7f163c4a11628f55a4df523b3ef

COINGECKO_URL=https://api.coingecko.com
COINGECKO_PAGES=4
COINGECKO_PER_PAGE=200
COINGECKO_TTL=60
```

### 3. Запуск

```bash
go run cmd/app/main.go
```

После запуска приложение подключается к Ethereum WebSocket и начинает обработку событий в режиме реального времени.
